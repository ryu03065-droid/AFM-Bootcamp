# 2강 (2026-06-13) — 프로젝트 관리 & Cursor/Claude Code 셋업

## 프로젝트 폴더 관리
- 프로젝트 폴더를 잘 관리하는 것이 중요하다. 다운로드 폴더를 작업 공간으로 사용.
- 프로젝트별로 파일을 정리하고, **프로젝트 폴더는 하나로 통합**해서 관리한다.

## Cursor & Claude Code
- Cursor는 파일 뷰어 겸 에디터로 사용.
- Cursor에서 Claude Code를 **켤 때는 토큰을 소모하지 않는다.** 채팅(대화)을 할수록 토큰이 소모된다.
- Cursor에서 Claude Code를 켜는 단축키: `Cmd + K`
- 여러 줄 입력 시: `\` (역슬래시) + Enter → 다음 줄로 줄바꿈.
- "보고서를 마크다운 파일로 만들어줘" 같은 요청이 유용하다.
- `.claude`에 에이전트를 추가한 뒤에는 `/reload-plugins`를 실행해야 반영된다.
- 서버가 없는 정적 클라이언트 사이트를 만들 때는 전용(리액트) 에이전트를 호출해서 사용한다.

## 앱 구조에 대한 관점
- AI를 활용해서 앱을 만들지만, **중요한 선택은 항상 사람이 한다.**
- 클라이언트는 React로 만든다.
  - AI가 React 코드를 학습 데이터로 많이 봤기 때문에 React를 특히 잘 다룬다.
  - 컴포넌트 재사용이 쉬워 프론트엔드 개발자들이 널리 사용하는 이유이기도 하다.

## Git & GitHub (서로 다른 개념)
- **Git**: 로컬에서 버전을 관리하는 도구.
  - `init` — 프로젝트(폴더) 단위로 Git 저장소를 초기화하는 명령.
  - `commit` — 저장 포인트(세이브 포인트). 피그마의 히스토리 저장 기능과 유사한 개념.
- **GitHub**: 원격 저장소 서비스. 무료로 사용 가능하며 AI 도구와 연동하기 쉽다.
  - `gh` CLI로 터미널에서 GitHub 조작 가능.
  - `push` — 로컬 커밋 내역을 GitHub 저장소로 동기화(업로드)하는 것.
