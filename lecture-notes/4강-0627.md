# 4강 (2026-06-27) — 에이전트/컨텍스트 심화 & 데이터베이스 기초

## Claude Code 실행 옵션
- `claude --dangerously-skip-permissions --effort medium` — 승인 스킵 + 작업 강도(effort) 지정.
- API 키에는 **퍼블릭 키**와 **시크릿 키** 두 종류가 있다는 점 확인.

## 메인 에이전트 / 서브 에이전트
- 서브에이전트 기능은 계속 업데이트되고 있음. 단순한 작업은 스킬만으로도 처리 가능.
- 메인 에이전트와 서브 에이전트가 **병렬로 작업**을 진행할 수 있다.
- 서브에이전트를 사용해도 토큰 사용량 자체는 동일하다 — 에이전트 두 개가 함께 일하는 방식일 뿐, 공짜로 늘어나는 게 아니다.

## 컨텍스트(Context) 개념
- 컨텍스트 = 문맥, 상황, 배경 정보.
- 컨텍스트 기반 = 앞뒤 상황을 이해하고 그에 맞게 판단하는 것.
- **모든 문맥은 텍스트로 변환 가능하다.** (나에 대한 이야기, 회사에 대한 이야기 등도 텍스트 문서로 만들어 컨텍스트로 활용 가능)
- 현재 시점에서 에이전트를 가장 잘 활용할 수 있는 도구는 Claude Code뿐이라는 언급.
- `/reload-plugins` — 에이전트를 새로 만들거나 수정한 뒤에는 반드시 실행해서 반영해야 한다.
- 컨텍스트로 쓰는 md 파일이 계속 업데이트될 예정이라면, 에이전트에게 "이 파일은 계속 업데이트될 거니까 매번 다시 읽고 답변해줘"라고 명시적으로 알려주는 것이 중요하다.
- 서버 쪽 코드는 로컬 파일을 전부 읽을 수 있는 권한을 가지고 있다는 점 유의.

## 데이터베이스 기초
- 가장 기본적인 형태의 데이터베이스는 문서 파일(예: 텍스트, JSON)이라고 볼 수 있다.
- **파싱(Parsing)**: 데이터를 읽어서 이해하기 쉬운 형태로 바꾸는 과정.
- 문서 파일을 그대로 DB로 쓰지 않는 이유: 컴퓨터가 파싱하기 어렵기 때문.
- 서버는 강력한 기능과 권한을 가지고 있다 (파일 읽기/쓰기 등을 좌지우지할 수 있음).
- 개발에서는 **CRUD**(Create, Read, Update, Delete)만 알아도 대부분의 기능을 구현할 수 있다.
- 데이터베이스는 문서 파일보다 더 규격화·구조화되어 있어서 앱과 AI 모두 읽고 다루기 쉽다. 항상 켜져 있어야(상시 구동) 한다는 특징이 있다.
- `.json` 파일, 로컬스토리지도 간단한 저장 수단으로 언급됨.

## Supabase (DB)
- Supabase는 무료로 사용할 수 있는 PostgreSQL 기반 DB 서비스.
- 무료 플랜에서는 프로젝트를 최대 2개까지 만들 수 있다.
- 구조: **웹서버(Vercel) — DB(Supabase)** 형태로 연결해서 사용.
- Connection string은 DB 서버에 접속하기 위한 주소 값이다 (민감 정보이므로 코드/문서에 하드코딩하지 않는다).

## 환경 변수 (.env)
- 로컬 개발환경용 `.env`와 Vercel 등 배포환경용 환경변수를 각각 따로 설정해야 한다.

> 참고: 원본 메모에 있던 실제 Supabase DB 비밀번호와 Connection string 값은 보안상 이 정리본에는 포함하지 않았습니다. 이미 노출된 값이라면 비밀번호를 재발급받으시길 권장드립니다.
