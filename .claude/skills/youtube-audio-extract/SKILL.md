# YouTube 영상 다운로드 + 음원 추출

유튜브 URL 하나로 영상(mp4)을 다운로드하고, 그 영상에서 음원(mp3)을 추출해 지정한 폴더에 함께 저장하는 스킬입니다.

## 준비물 확인

`yt-dlp`와 `ffmpeg`가 있어야 합니다. 없으면 Homebrew로 설치하세요 (실행 전 사용자에게 설치해도 되는지 확인):

```bash
which yt-dlp ffmpeg
# 없으면
brew install yt-dlp ffmpeg
```

## 작업 흐름

### 1단계: URL과 저장 폴더 확인
- 사용자가 유튜브 URL을 주지 않았다면 물어보세요.
- 저장할 폴더도 사용자가 매번 지정합니다 (기본값을 임의로 만들지 마세요). 폴더가 없으면 생성하세요.

```bash
mkdir -p "<저장할 폴더 경로>"
```

### 2단계: 영상 다운로드 (mp4)

지정한 폴더 안에서 실행합니다. 영상 제목이 파일명이 되며, 특수문자(`/` 등)가 포함될 수 있으니 항상 큰따옴표로 감싸서 다루세요.

```bash
cd "<저장할 폴더 경로>" && yt-dlp -f "bestvideo[ext=mp4]+bestaudio[ext=m4a]/best[ext=mp4]/best" -o "%(title)s.%(ext)s" "<유튜브 URL>"
```

다운로드가 끝나면 출력 로그의 `Merging formats into "..."` (또는 마지막 `Destination:`) 줄에서 실제로 저장된 파일명을 확인하세요. 유튜브 원본 제목은 대괄호 태그, 이모지, 화질 표시 등이 잔뜩 붙어 지저분한 경우가 많으므로, 다음 단계에서 바로 정리합니다.

### 3단계: 파일 이름 정리

다운로드된 원본 제목에서 핵심만 남기고 나머지는 제거해 깔끔한 파일명으로 바꿉니다.

- 제거 대상: 대괄호/괄호 태그(`[4K]`, `[한/영 자막]`, `(Official Video)` 등), 이모지, 느낌표·물음표 같은 클릭베이트성 문장부호, "역대급", "레전드" 같은 수식어, 중복 공백
- 남길 것: 아티스트/채널명과 곡·영상의 핵심 제목, 그리고 맥락상 의미 있는 정보(예: "Live Aid Concert", "커버", "풀영상")가 있다면 " - " 나 괄호로 간단히 붙여서 유지
- 예시: `역대급 최고의 공연! [한/영 자막] Queen(퀸) 의 'Radio GaGa' Live aid concert.mp4` → `Queen - Radio Gaga (Live Aid Concert).mp4`
- 파일시스템에 문제되는 문자(`/`, `:`, `*`, `?`, `"`, `<`, `>`, `|`)는 정리 과정에서 자연스럽게 빠지지만, 혹시 남아있다면 제거하거나 `-`로 바꾸세요
- 애매하면 과도하게 축약하지 말고, 원래 제목에서 유의미한 정보는 최대한 보존하세요

정리한 이름으로 mp4 파일명을 바꿉니다 (실제 원본/정리된 이름으로 치환):

```bash
cd "<저장할 폴더 경로>" && mv "<원본 다운로드 파일명>.mp4" "<정리된 이름>.mp4"
```

### 4단계: 음원 추출 (mp3)

정리된 이름의 mp4에서 오디오 트랙만 뽑아 같은 폴더에 같은 이름의 mp3로 저장합니다. 원본 mp4는 삭제하지 않고 그대로 둡니다 (영상+음원 둘 다 보관이 기본 동작).

```bash
cd "<저장할 폴더 경로>" && ffmpeg -y -i "<정리된 이름>.mp4" -vn -acodec libmp3lame -q:a 2 "<정리된 이름>.mp3"
```

### 5단계: 결과 확인 및 보고
- `ls -la "<저장할 폴더 경로>"`로 mp4/mp3가 모두 생성됐는지 확인하세요.
- 저장된 두 파일의 경로와 대략적인 용량을 사용자에게 알려주세요.

## 참고
- mp3 비트레이트는 `-q:a 2`(가변 비트레이트, 약 190kbps 수준의 고음질)를 기본으로 사용합니다. 사용자가 특정 화질/음질을 요청하면 `-f` 포맷 선택자나 `-q:a` 값을 조정하세요.
- 재생목록 URL이 오면 여러 영상이 한 번에 받아지므로, 사용자가 특정 영상 하나만 원하는지 먼저 확인하는 게 안전합니다.
