# Interest Research Skill

매번 반복되는 "웹 리서치 → md 정리 → 노션 공유" 흐름을 하나로 묶은 스킬입니다. 주제만 던지면 항상 같은 포맷으로 정리하고, 항상 같은 곳(노션)에 공유합니다.

## 작업 흐름

### 1단계: 주제 파악
- 사용자가 리서치할 주제를 이미 말했다면 그대로 사용하세요.
- 주제가 불명확하면 무엇을 리서치할지 먼저 물어보세요 — 이 스킬은 주제를 추측해서 진행하지 않습니다.
- 검색 키워드는 한국어 표현이라도 영어 키워드를 함께 섞어서 검색하세요 (영어 자료가 더 풍부한 경우가 많음).

### 2단계: 웹 리서치
- `WebSearch`로 최소 2개 이상의 서로 다른 쿼리로 검색하세요 (예: 핵심 주제 + 인접 주제/최신 동향).
- 결과에서 근거가 될 만한 출처를 3~6개 정도 골라 핵심 포인트를 뽑아내세요.
- 특정 결과를 더 깊이 파고들 필요가 있으면 `WebFetch`로 원문을 확인하세요.
- 각 핵심 포인트는 반드시 출처 링크와 함께 남기세요. 출처 없는 주장은 쓰지 마세요.

### 3단계: 마크다운으로 정리
- 저장 위치는 **스킬을 실행한 현재 작업 디렉토리(`pwd`) 기준** `research/` 폴더입니다. 없으면 생성하세요.
- 파일명: `research/{topic-slug}_{YYYY-MM-DD}.md` (topic-slug는 영문 소문자+하이픈)
- 아래 구조를 그대로 따르세요:

```markdown
# {주제} 리서치 ({YYYY-MM-DD})

## 한 줄 요약
{전체 내용을 한두 문장으로}

## 핵심 트렌드 / 포인트
1. **{포인트 제목}** — {설명 2~3문장} ([출처]({URL}))
2. **{포인트 제목}** — {설명 2~3문장} ([출처]({URL}))
...

## 적용 아이디어
- {이 리서치를 실제로 어떻게 활용할지 1~3개}

## 출처
- [{사이트/글 제목}]({URL})
- [{사이트/글 제목}]({URL})
...
```

### 4단계: 노션에 공유
- 먼저 `mcp__notion__notion-search`로 관련 기존 페이지(예: 리서치 아카이브, 같은 주제의 이전 문서)가 있는지 확인하세요.
- 적절한 부모 페이지가 있으면 그 아래에, 없으면 워크스페이스 최상위(부모 없이) private 페이지로 생성하세요.
- 페이지 생성 전 반드시 MCP 리소스 `notion://docs/enhanced-markdown-spec`를 (MCP 리소스 읽기 인터페이스로) 한 번 읽어 Notion 마크다운 문법을 확인하세요. 문법을 추측하지 마세요.
- `mcp__notion__notion-create-pages`로 페이지를 생성하세요.
  - title: `[리서치] {주제} ({YYYY-MM-DD})`
  - icon: 주제와 어울리는 이모지 1개
  - content: 3단계에서 만든 마크다운 본문 (제목은 properties.title에만 넣고 본문에 중복하지 마세요)
- 노션 연결이 안 되어 있거나 페이지 생성이 실패하면, 에러를 그대로 사용자에게 전달하고 로컬 md 파일은 이미 만들어졌음을 알려주세요. (실패해도 3단계 결과물은 항상 남아있어야 합니다)

### 5단계: 완료 보고
- 저장된 로컬 md 경로와, 생성된 노션 페이지 URL을 함께 알려주세요.

## 중요 규칙
1. 출처 없는 트렌드/주장은 쓰지 마세요 — 모든 핵심 포인트에 링크를 답니다.
2. 로컬 md 파일은 노션 공유 성공 여부와 무관하게 항상 `research/`에 저장되어야 합니다.
3. 노션 문법은 추측하지 말고 `notion://docs/enhanced-markdown-spec` 리소스를 확인한 뒤 작성하세요.
4. 이미 노션에 같은 주제의 최근 리서치가 있다면, 새로 만들기 전에 사용자에게 알려주고 이어서 진행할지 확인하세요.
